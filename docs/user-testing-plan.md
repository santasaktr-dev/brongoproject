# BRONGO Mix Lab — MVP User Testing Plan

## เป้าหมาย

ทดสอบว่าผู้ปกครองเข้าใจวิธีเล่น ทำ flow จนจบ เข้าใจความหมายของคะแนน และแยกแยะได้ว่าเกมให้ความรู้เรื่องส่วนผสม ไม่ใช่เครื่องมือเลือกยาหรือวินิจฉัยโรค

## Participants

- รอบแรก: ผู้ปกครอง 5–8 คน
- มีบุตรอายุประมาณ 1–8 ปี
- อย่างน้อยครึ่งหนึ่งเคยซื้อหรือให้ยาน้ำแก่บุตร
- ทดสอบบนโทรศัพท์ของผู้เข้าร่วมเมื่อทำได้
- เด็กเข้าร่วมดูหรือช่วยแตะหน้าจอได้ แต่คำตอบด้านความเข้าใจเก็บจากผู้ปกครอง
- ไม่เก็บชื่อเด็ก ประวัติการรักษา โรค หรือยาที่ใช้อยู่

## Session Format

- 30–40 นาทีต่อคน
- Moderator 1 คน และ note taker 1 คนถ้าเป็นไปได้
- screen recording เฉพาะเมื่อได้รับความยินยอม
- ใช้วิธี think aloud แต่ไม่บังคับให้พูดตลอดเวลา

## Moderator Opening Script

`วันนี้เรากำลังทดสอบเว็บไซต์ ไม่ได้ทดสอบคุณ บางจุดอาจยังไม่สมบูรณ์ ขอให้ลองใช้ตามธรรมชาติ หากสงสัยให้พูดออกมาได้ แต่ระหว่างภารกิจฉันอาจยังไม่ตอบทันที เพื่อดูว่าเว็บไซต์อธิบายตัวเองได้หรือไม่ กิจกรรมนี้ไม่ใช่คำแนะนำทางการแพทย์ และเราจะไม่ถามข้อมูลสุขภาพของคุณหรือบุตร`

## Pre-test Questions

1. ปกติคุณใช้โทรศัพท์ทำกิจกรรมหรืออ่านข้อมูลเกี่ยวกับสุขภาพบ่อยแค่ไหน?
2. เมื่อเห็นเว็บไซต์เกี่ยวกับยาสำหรับเด็ก อะไรทำให้คุณรู้สึกว่าเว็บไซต์น่าเชื่อถือ?
3. เคยเล่นกิจกรรม interactive หรือ quiz จากแบรนด์หรือไม่?

## Test Tasks

### Task 1: Understand the Landing Page

Prompt: `ลองดูหน้าแรก แล้วบอกว่าคุณคิดว่าเว็บไซต์นี้ให้ทำอะไร`

Observe:

- เข้าใจว่าเป็นเกมเลือกส่วนผสมหรือไม่
- มองเห็น CTA ภายใน 5 วินาทีหรือไม่
- เข้าใจว่าเกี่ยวข้องกับ BRONGO หรือไม่

ห้าม moderator อธิบายวิธีเล่นก่อนผู้ใช้กดเริ่ม

### Task 2: Create a Mix

Prompt: `เลือกส่วนผสมตามที่คุณสนใจอย่างน้อย 3 รายการ แล้วดูผลลัพธ์`

Observe:

- เข้าใจ selected state หรือไม่
- พบทั้ง 5 หมวดหรือไม่
- เข้าใจ Vitamin B6/ไม่ใส่หรือไม่
- เห็นว่าปุ่มดูผลยัง disabled ตอนยังไม่เลือกหรือไม่
- มี accidental taps หรือย้อนกลับโดยไม่ตั้งใจหรือไม่

### Task 3: Explain the Score

Prompt หลังเห็นผล: `คะแนนนี้หมายความว่าอย่างไรในความเข้าใจของคุณ?`

Pass condition:

- เข้าใจว่าเป็นสัดส่วนของรายการที่เลือกแล้วตรงกับ BRONGO
- ไม่ตีความว่าเป็นความปลอดภัย ประสิทธิภาพ หรือคำแนะนำการใช้ยา
- สำหรับ 100% ต้องเข้าใจว่าอาจไม่ได้เลือกครบสูตร

### Task 4: Find Ingredient Information

Prompt: `ลองหาว่าส่วนผสมไหนมีใน BRONGO และเลือกอ่านเรื่องน่ารู้หนึ่งรายการ`

Observe:

- แยก matched/unmatched ได้โดยไม่พึ่งสีอย่างเดียว
- พบ fun fact
- เข้าใจว่าบางรายการไม่มี fun fact

### Task 5: Edit the Mix

Prompt: `กลับไปแก้สูตรเดิม โดยเปลี่ยนอย่างน้อยหนึ่งรายการ แล้วดูผลอีกครั้ง`

Observe:

- หา CTA แก้สูตรเจอ
- สูตรเดิมยังอยู่
- เข้าใจว่าคะแนนเปลี่ยนเพราะ denominator/matches เปลี่ยน

### Task 6: Find Product Safety Information

Prompt: `หาข้อมูลว่าอาการแบบใดต้องหยุดยาและพบแพทย์ทันที`

Observe:

- เข้า BRONGO Detail ได้
- ใช้ accordion ได้
- พบหัวข้อภายใน 30 วินาที
- แยกข้อมูลเกมออกจากข้อมูลยาได้

## Post-test Questions

1. ส่วนไหนสนุกที่สุด เพราะอะไร?
2. ส่วนไหนสับสนหรือทำให้ลังเล?
3. คะแนนหมายความว่าอย่างไร?
4. หลังเล่น คุณคิดว่าเว็บไซต์กำลังแนะนำให้เลือกยาหรือไม่ เพราะอะไร?
5. ข้อมูลส่วนผสมอ่านง่ายเพียงใด ให้คะแนน 1–5
6. ภาพและ animation ดูเหมาะกับผู้ปกครองและเด็กหรือไม่?
7. มีข้อความใดที่ฟังดูเป็นการโฆษณามากเกินไปหรือไม่น่าเชื่อถือหรือไม่?
8. คุณอยากเล่นใหม่หรือส่งให้ผู้ปกครองคนอื่นหรือไม่?

## Metrics

| Metric | MVP target |
|---|---:|
| Landing → Start completion | ≥ 90% |
| Start → Result completion | ≥ 85% |
| เข้าใจคะแนนถูกต้องหลังอ่านหน้า Result | ≥ 80% |
| ไม่ตีความว่าเกมวินิจฉัย/เลือกยาให้ | ≥ 90% |
| พบ safety section ภายใน 30 วินาที | ≥ 80% |
| Accidental tap rate ต่อ session | ≤ 2 ครั้ง |
| Usability rating | ≥ 4/5 median |

## Severity Classification

- Critical: ทำ flow ต่อไม่ได้ ข้อมูลยาผิด หรือทำให้เข้าใจว่าเป็นคำแนะนำทางการแพทย์
- High: ผู้ใช้ส่วนใหญ่ตีความคะแนนผิด หา safety information ไม่พบ หรือ action สำคัญใช้ไม่ได้
- Medium: สับสนชั่วคราว ต้องลองมากกว่าหนึ่งครั้ง แต่ทำงานสำเร็จ
- Low: wording, spacing หรือ polish ที่ไม่ขัดขวางงาน

## Release Rule

- Critical ต้องเป็นศูนย์
- High ต้องแก้ก่อนรอบทดสอบถัดไป
- Medium จัดลำดับตามความถี่และผลกระทบ
- Low รวมไว้ใน polish backlog
- ต้องทำรอบทดสอบซ้ำอย่างน้อย 3–5 คนหลังแก้ Critical/High
